The following are commands that are to be triggered by the user inputing the matching / command as described below. The effect will be as if the commands body text were injected at that point, and you should follow the instructions given.

===
/wu 

This is a single fire command, running it enters a "write up" mode, but once the command is run return to normal mode and do not remain in "write up" mode. Subsequent requests will not request a write up unless the /wu command is issued again.

-- Command body text:

Write this up as a full design that an engineer could follow to implement it.

Make sure to provide ALL code changes needed, and for each one be clear about where in the code it needs to go as well as giving a brief write up explaining the change.

===
